# CRIAÇÃO DE IDENTIDADE VISUAL PARA LANDING PAGES A PARTIR DE REFERÊNCIAS

## O QUE ESTE PROMPT FAZ
Você vai criar uma identidade visual ORIGINAL para uma landing page. Não é trocar cores de um template. É criar uma experiência de scroll onde cada seção é um ATO NARRATIVO — com ritmo, tensão, clímax e resolução visual.

O resultado é um documento .md que funciona como "DNA visual" da landing page — qualquer IA que ler esse documento vai gerar uma página com personalidade própria, não um template recolorido do Framer.

---

## O PROBLEMA QUE ESTE PROMPT RESOLVE

Quando você pede pra uma IA criar uma landing page, ela entrega o DEFAULT:
Hero centralizado + headline genérica + botão CTA + 3 cards de features lado a lado + seção de depoimentos com aspas + pricing table + footer. Tudo simétrico, limpo, morto. O scroll não tem ritmo. Cada seção é uma caixa isolada. Nada conecta. Nada surpreende.

Se você pede "com identidade visual", ela troca as cores. Talvez mude a fonte. Talvez adicione um gradiente no hero. Mas a ESTRUTURA narrativa continua sendo a mesma: seções empilhadas como slides de PowerPoint. Não tem RITMO.

E se você dá uma "biblioteca de decorações genéricas" (blobs, dot grids, partículas, gradientes mesh), ela espalha esses elementos em tudo. O resultado: seções que antes eram limpas agora têm barulho visual genérico. Um blob atrás do headline não é identidade — é ruído. Partículas flutuando no hero não é personalidade — é enfeite que toda LP de SaaS tem.

E se você pede "identidade com cores", ela cria gradientes em todos os fundos, glow em todo botão, 5 cores pra categorizar features, sombras coloridas em todo card. O resultado: arco-íris visual que parece psicodélico, não profissional. Linear não usa 5 cores. Usa violeta + neutros. Stripe não usa gradientes em tudo. Usa gradientes PONTUAIS com propósito.

**O que falta não é decoração. É DRAMATURGIA VISUAL.**

Compare mentalmente:
- Uma seção de features com 3 cards lado a lado com ícone + título + descrição → TODO TEMPLATE DO MUNDO
- Uma seção de features onde cada feature é uma CENA: a feature de "automação" mostra um fluxo de nós conectados com dados fluindo entre eles em tempo real; a feature de "analytics" mostra um dashboard minimalista com um gráfico que se desenha conforme o scroll → CONCEITO NARRATIVO

- Um hero com headline centralizada + subtítulo + botão + mockup do app → TODO TEMPLATE DO MUNDO
- Um hero onde o headline está à esquerda, e à direita uma composição visual SVG que CONTA A HISTÓRIA do produto — não um screenshot, mas uma CENA CONCEITUAL que traduz o que o produto faz → DISRUPTIVO

- Uma seção "How it Works" com 3 passos numerados → TODO TEMPLATE DO MUNDO
- Uma seção "How it Works" onde os passos são conectados por uma LINHA NARRATIVA visual — um path SVG que serpenteia pela seção, passando por cada step como uma jornada, com animação de scroll que revela o caminho conforme o usuário desce → CONCEITO NARRATIVO

A diferença: **decoração genérica** é visual que poderia estar em qualquer LP de qualquer produto. **Conceito narrativo** é uma composição visual que SÓ faz sentido naquela seção daquela LP porque conta uma história sobre o que o produto RESOLVE.

## A DIFERENÇA ENTRE DECORAÇÃO E DRAMATURGIA

### DECORAÇÃO GENÉRICA (evitar como estratégia principal):
- Gradient mesh no hero (toda LP de SaaS em 2024)
- Dot grids no fundo (toda LP do Framer)
- Partículas flutuantes (toda LP de crypto)
- Blobs coloridos atrás de headlines
- Glow borders em todo hover
- Glassmorphism em todos os cards
- Gradientes em todas as seções
- Mockup do app como hero image (preguiça visual)

Esses elementos têm seu lugar — como **camada ambiente**, como **textura de fundo**, como **complemento**. Mas eles NUNCA devem ser a principal estratégia de impacto. Se toda seção da LP tem blob + gradiente + partículas, não é identidade — é um template Framer recolorido.

### DRAMATURGIA VISUAL (o objetivo):
Cada seção importante tem uma **composição visual narrativa** que conta uma história sobre o que aquela seção COMUNICA. A LP inteira tem um RITMO de scroll — alternância de tensão/respiro, densidade/espaço, escuro/claro, texto/visual. O visitante não só LÊ a página — ele EXPERIMENTA uma jornada.

**Como funciona na prática:**

O hero do Vercel Ship não tem um mockup. Tem uma **composição tipográfica massiva** com a palavra "SHIP" dominando a tela, sobre um fundo dark com gradientes sutis que criam profundidade cósmica. É um conceito: "magnitude, escala, ambição". Não precisa de screenshot — a tipografia É o visual.

O hero do Linear não tem um blob. Tem uma **interface do próprio produto** renderizada em perspectiva isométrica, com linhas de conexão entre elementos, sobre um fundo dark com um grid sutil. É um conceito: "ordem no caos, precisão, ferramenta de poder".

A seção de features do Stripe não tem 3 cards lado a lado. Tem **composições visuais únicas para cada feature** — diagramas, ilustrações de fluxo, representações visuais do que cada API endpoint FAZ. Cada feature tem sua própria narrativa visual.

A seção "How it Works" da Resend não tem 3 passos numerados em caixas. Tem **um fluxo visual** com snippets de código reais + setas + resultados — mostrando LITERALMENTE como funciona, não descrevendo com texto genérico.

**PRINCÍPIO FUNDAMENTAL: Cada seção importante merece um CONCEITO NARRATIVO, não uma DECORAÇÃO.**

## A REGRA DA COR ÚNICA

Olhe para qualquer LP com identidade forte:

- **Linear:** fundo dark + **violeta/azul**. Só violeta. Todo o resto é cinza e branco. A cor aparece nos acentos, nos gradientes pontuais, nos elementos de UI, nos hovers.
- **Vercel:** fundo dark + **branco**. Sem cor accent sequer. Preto e branco puro + gradientes sutis ocasionais. A identidade vive na tipografia e no ritmo.
- **Resend:** fundo dark + **branco**. Um toque de cor nos exemplos de código. A identidade vive na clareza brutal.
- **Supabase:** fundo dark + **verde**. Só verde. Todo o resto é escuro com cinza. O verde é cirúrgico — botões, linhas de código, ícones ativos.
- **Stripe:** fundo light/dark alternado + **violeta**. Os gradientes são violeta-para-azul, não arco-íris. UMA família de cor.
- **Cal.com:** fundo light + **preto forte**. Sem cor accent visível. A força vem da tipografia e dos contrastes.

**O padrão é claro:** LP com identidade forte = base limpa + UMA cor primária (ou até nenhuma cor extra) + dramatúrgia visual nas seções.

### O que NÃO é identidade:
- Gradientes coloridos em todas as seções → é barulho
- 5 cores pra categorizar features → é arco-íris
- Glassmorphism em tudo → é efeito de template
- Glow colorido em todo botão/card → é noise

### A regra:
1. **Escolha UMA base:** light ou dark
2. **Escolha UMA cor primária forte** (ou NENHUMA extra — identidade pode ser tipográfica/estrutural)
3. **Todo o resto é neutro:** cinzas, brancos, pretos
4. **Gradientes, se existirem,** são variações da cor primária + neutros — não arco-íris
5. **A cor primária aparece com INTENÇÃO:** CTAs, acentos, highlights — não em tudo

## AS TRÊS CAMADAS DA IDENTIDADE DE UMA LP

### Camada 1: RITMO E ESTRUTURA
Como a página se desenrola no scroll.
↳ Ritmo de scroll (alternância de seções densas/respiradas, escuras/claras, cheias/vazias)
↳ Layout por seção (hero fullscreen? split? texto-esquerda-visual-direita? centralizado? assimétrico? grid-breaking?)
↳ Hierarquia de informação (o que aparece primeiro? o que é revelado depois? como o argumento se constrói?)
↳ Transições entre seções (corte seco? overlap? gradiente de fundo? elemento visual que "conecta" seções?)
↳ Navegação (nav fixa? transparente? que aparece no scroll? minimalista? robusta?)
↳ Breakpoints narrativos (onde o visitante PARA e absorve? onde ele é puxado pra continuar scrollando?)

### Camada 2: LINGUAGEM VISUAL
Como os elementos se expressam.
↳ Tipografia como ARMA (uma fonte hero impactante? mistura intencional? tamanhos extremos? tight letter-spacing? oversized headlines?)
↳ Cores como sistema (UMA cor accent + neutros. Ver regra acima)
↳ Geometria (cantos, formas, ângulos — o que é pill? o que é reto? o que é cortado em diagonal?)
↳ Profundidade (flat? sombras dramáticas? layers sobrepostos? parallax?)
↳ Micro-interações (hover states que surpreendem, elementos que reagem ao scroll, cursores custom)
↳ Fotografia/Visual Direction (se usar fotos: qual tratamento? duotone? com overlay? em containers customizados?)

### Camada 3: DRAMATURGIA VISUAL ← A CAMADA QUE FAZ A LP SER DISRUPTIVA
O que vive DENTRO de cada seção além de texto e botões.

Essa é a camada que separa uma LP memorável de um template bonito.

**Dramaturgia visual tem dois níveis:**

#### Nível A: ATMOSFERA (fundo, ambiente)
São elementos sutis que criam o CLIMA da página. Vivem no fundo, nunca são protagonistas. Exemplos:
- Pattern geométrico no background (grid, topografia, circuito, constelação) — opacity 3-8%
- Gradiente ambiental sutil que muda conforme o scroll (dark → slightly lighter → dark)
- Noise/grain como textura (dá sensação de materialidade, remove o "clean demais" dos templates)
- Elementos wireframe decorativos nas margens (como Firecrawl: snippets de código, wireframes)

Esses elementos são monocromáticos, usam APENAS neutros ou a cor accent em opacidade muito baixa. NUNCA gradientes coloridos como atmosfera — isso vira barulho de template.

#### Nível B: COMPOSIÇÕES NARRATIVAS (dentro das seções) ← O QUE TORNA A LP DISRUPTIVA
São composições visuais ÚNICAS para cada seção importante. Cada uma conta uma história visual sobre o que aquela seção COMUNICA ao visitante. Não são ícones grandes. Não são mockups. São CENAS que traduzem um conceito.

**A IA que implementa deve CRIAR essas composições usando SVG inline, CSS, animações de scroll, e composição de formas.** Não são assets externos (na maioria). São composições originais que juntas formam uma cena com significado.

**O que a IA consegue criar em SVG/CSS puro:**
↳ Diagramas de fluxo animados (nós conectados com dados "fluindo")
↳ Interfaces simplificadas do produto (representação abstrata, não screenshot)
↳ Cenas com linhas e nós (rotas, constelações, grafos, timelines de processo)
↳ Composições tipográficas impactantes (palavras como elemento visual, não só texto)
↳ Elementos geométricos temáticos (formas que representam conceitos do produto)
↳ Gráficos/dashboards estilizados (versão artística dos dados, não chart.js)
↳ Paths animados com scroll (linhas que se desenham conforme o scroll)
↳ Comparações visuais before/after (lado a lado com tratamentos visuais diferentes)
↳ Cenários mínimos (horizonte, paisagem abstrata, representação de "mundo" do produto)

**O que funciona COM AJUDA (humano gera, IA integra):**
↳ Screenshots reais do produto (humano tira, IA estiliza com sombras, perspectiva, containers)
↳ Fotos (humano fornece ou pega de Unsplash, IA integra com tratamento visual)
↳ Vídeos de demo (humano grava, IA posiciona com autoplay/loop)
↳ Logos de clientes para social proof (humano fornece, IA organiza)
↳ Ícones/mascotes complexos (humano gera, IA integra)

**O que NÃO funciona (IA gera mal, evite):**
↳ Pessoas, rostos, mãos via código
↳ Mockups 3D realistas de devices
↳ Ilustrações com muitos detalhes figurativos

## COMO PENSAR COMPOSIÇÕES NARRATIVAS PARA CADA SEÇÃO (PROCESSO OBRIGATÓRIO)

Para cada seção importante da LP, a IA que gera o documento de identidade deve seguir este processo:

### Passo 1: O que esta seção COMUNICA na narrativa?
Não "o que ela contém" — qual é o ARGUMENTO que ela faz ao visitante.
- "Hero" → a promessa principal: o que o produto FAZ e por que importa
- "Social Proof" → validação: outras pessoas confiam
- "Features" → prova da promessa: COMO o produto entrega o que prometeu
- "How it Works" → remoção de atrito: é simples de usar
- "Testimonials" → prova social emocional: pessoas reais tiveram resultado
- "Pricing" → decisão: qual opção é pra mim?
- "CTA Final" → urgência/desejo: quero agora

### Passo 2: Qual COMPOSIÇÃO VISUAL traduz esse argumento?
Não pense em "decoração pra essa seção". Pense: **qual cena visual REFORÇA o argumento?**
- Promessa → composição que MOSTRA o resultado (não conta — mostra)
- Validação → logos tratados com elegância, não jogados em fila
- Prova → cenas conceituais ÚNICAS por feature (não ícone+texto genérico)
- Simplicidade → fluxo visual que demonstra o processo (não 3 caixas numeradas)
- Resultado → quotes integradas ao visual, não cards com aspas
- Decisão → tabela que guia o olhar, não pricing table padrão

### Passo 3: Essa composição é implementável em SVG/CSS/animação de scroll?
- Se SIM → descreva com detalhe suficiente
- Se NÃO → simplifique: versão geométrica/wireframe
- Se PRECISA DE ASSET → marque como [ASSET EXTERNO] e explique

### Passo 4: Descreva a cena visual COM DETALHES
Não diga "adicione um visual impactante no hero". Descreva a COMPOSIÇÃO:
- "Hero: split layout. Headline à esquerda em fonte display, tamanho massivo (64-80px), com 2-3 palavras-chave na cor accent. À direita, uma composição SVG mostrando um grafo de nós interconectados — 5 nós principais com labels simplificados (ícones, não texto), conectados por linhas curvas com dash-animation, representando dados fluindo entre sistemas. Os nós pulsam sutilmente (opacity 0.7→1.0 em loop de 3s, staggered). A composição tem opacity 90% e usa apenas accent-primary + neutros. No mobile, a composição vai acima do headline em escala reduzida."

Esse nível de detalhe gera resultado. "Adicione algo visual" gera template.

## A REGRA DO RITMO DE SCROLL

Uma LP disruptiva não é uma sequência de seções bonitas — é uma **experiência ritmada** onde cada seção tem um papel narrativo e uma ENERGIA diferente.

### O conceito de ritmo:
- **Seção DENSA** (hero com composição visual, muita informação, impacto) → seguida de...
- **Seção RESPIRO** (social proof com logos em fila limpa, espaço negativo, simplicidade) → seguida de...
- **Seção DENSA** (features com composições narrativas, muito conteúdo visual) → seguida de...
- **Seção RESPIRO** (quote única de depoimento em tela cheia, tipografia grande, espaço) → seguida de...
- **Seção CLÍMAX** (CTA final com energia máxima)

### Ferramentas de ritmo:
- **Alternância de fundo:** dark → light → dark cria "atos" visuais
- **Alternância de densidade:** seção cheia → seção vazia → seção cheia
- **Alternância de layout:** centralizado → assimétrico → centralizado
- **Escala tipográfica:** headline 80px → texto 18px → quote 48px → texto 16px
- **Espaço negativo como arma:** uma frase sozinha com 200px de padding em cima e embaixo tem mais impacto que uma seção inteira de features

### O erro comum:
Toda seção com a mesma energia = monotonia. Toda seção com alta energia = exaustão. O segredo é CONTRASTE.

## REGRAS TÉCNICAS (OBRIGATÓRIAS)

### Stack
- **Tailwind CSS** para toda estilização (NUNCA criar arquivos .css separados, exceto @keyframes se necessário)
- **shadcn/ui** como base de componentes quando aplicável, customizados via className
- **Tokens semânticos** para TODOS os valores visuais
- **Framer Motion** (ou CSS puro) para animações de scroll e micro-interações

### Tokens Semânticos
Todos os valores visuais devem ser definidos como tokens com nomes que carregam INTENÇÃO.

**Errado:** `bg-[#0A0A0F]`, `text-[80px]`, `rounded-[24px]`
**Certo:** `bg-surface-page`, `text-hero`, `rounded-card`

### Proibições
- NUNCA usar cores/radius/sombras padrão do Tailwind — apenas tokens do documento
- NUNCA entregar seções que sejam apenas "headline + subtítulo + botão + ícone" sem composição visual
- NUNCA usar 3 cards lado a lado como única estratégia de features — isso é template
- NUNCA usar mockup de iPhone/laptop como hero visual (a menos que o produto SEJA um app mobile)
- NUNCA usar decoração genérica (blobs + dots + partículas) como substituto de composição narrativa
- NUNCA criar múltiplas cores accent — UMA cor ou nenhuma extra
- NUNCA fazer todas as seções com a mesma energia — RITMO é obrigatório
- NUNCA usar gradientes em tudo como "identidade"
- NUNCA usar hero centralizado genérico se não houver uma razão narrativa forte

## ETAPA 2: CRIAÇÃO DA IDENTIDADE

Use os princípios extraídos + contexto do projeto.

### O teste central:
**"Tirando as cores, o que muda em relação a um template Framer/Webflow padrão?"**

A resposta deve incluir mudanças nas TRÊS camadas:
- Ritmo: o scroll tem alternância de energia, não seções uniformes
- Linguagem: tipografia/geometria/interações têm personalidade
- Dramaturgia: as seções têm COMPOSIÇÕES NARRATIVAS — não são blocos de texto com ícones

Se a resposta for "só mudaram as cores e os gradientes", está errado.
Se a resposta for "mudou layout mas as features são 3 cards com ícone", está errado.
Se a resposta for "tem blobs e partículas bonitas mas nenhuma cena conceitual", TAMBÉM está errado.

### O processo para dramaturgia visual:

**Para cada seção importante da LP, siga o processo de 4 passos:**

1. **O que esta seção COMUNICA?** (argumento na narrativa)
2. **Qual COMPOSIÇÃO VISUAL reforça esse argumento?** (cena, não decoração)
3. **É implementável em código puro?** (viabilidade)
4. **Descreva a COMPOSIÇÃO com detalhe** (pra IA implementar, não interpretar)

**Escolha as 4-6 seções mais importantes** e crie composições narrativas para cada uma. Seções intermediárias podem ser clean — o contraste entre seções ricas e seções respiradas É o ritmo.

### Regra de atmosfera:
ALÉM das composições por seção, defina UMA atmosfera global para a LP — um pattern, textura ou tratamento de fundo que cria o CLIMA geral. Essa atmosfera deve ser temática e com opacidade muito baixa (3-8%).

## REGRAS PARA A IA QUE GERA ESTE DOCUMENTO

1. **NARRATIVA > DECORAÇÃO.** Nunca sugira "adicione um blob no hero" ou "coloque partículas". Sempre pergunte: "Qual composição visual REFORÇA O ARGUMENTO desta seção?" e descreva essa composição.
2. **UMA COR, UMA MARCA.** A paleta DEVE ter apenas UMA cor accent forte (ou nenhuma extra). Não crie arco-íris. Gradientes em tudo não é identidade — é barulho.
3. **DESCREVA, NÃO CODIFIQUE.** O documento NÃO deve conter blocos de código. Descreva cenas com palavras detalhadas — a IA que implementa traduz em código. Código pronto vira template; descrição vira criação.
4. **4 PASSOS POR SEÇÃO.** Para cada seção importante: (1) O que COMUNICA, (2) Qual COMPOSIÇÃO VISUAL, (3) É viável em código?, (4) Descreva a CENA com detalhe.
5. **RITMO É OBRIGATÓRIO.** Defina a energia de cada seção. O scroll DEVE alternar entre denso/respiro, escuro/claro, visual/textual. LP sem ritmo é PowerPoint.
6. **TEMPERO SIMPLES MAS CONCEITUAL.** Um path SVG que se desenha no scroll conectando os steps de "How it Works" é simples E conceitual. Partículas flutuando no fundo é simples mas genérico. Prefira o primeiro.
7. **TIPOGRAFIA É ARMA.** Em LPs, a tipografia pode SER o visual. Uma headline de 80px com letter-spacing apertado e peso 800 não precisa de ilustração — ELA É a ilustração. Pense em tipografia como composição visual.
8. **DETALHE AS COMPOSIÇÕES.** "Adicione um visual legal no hero" é vago. "Split layout: headline à esquerda com fonte display 72px, line-height 0.95, 3 palavras por linha, a palavra-chave na segunda linha em accent-primary. À direita, composição SVG de 500x400px mostrando um grafo de 5 nós circulares (24px) conectados por linhas curvas (stroke 1.5px), com um nó central maior (36px) na cor accent pulsando. Os nós menores têm ícones simplificados dentro (code, database, api, analytics, deploy). As linhas de conexão têm dash-animation de 20s loop. Tudo em opacity 85%. No mobile: composição empilhada, headline em cima (48px), SVG embaixo em 300x250px com 3 nós apenas." ESSE nível gera resultado.
9. **Não assuma padrões.** Hero centralizado, 3 cards de features, pricing table — nada é obrigatório. Questione cada convenção.
10. **Tokens semânticos.** Nomes com intenção. Valor definido uma vez. No código, só o token.
11. **Escolha momentos.** Não decore tudo — escolha 4-6 seções de alto impacto. O contraste entre seções ricas e seções limpas É o ritmo.
12. **SCROLL-DRIVEN DESIGN.** Pense na LP como uma experiência de scroll, não como uma página estática. Elementos que se revelam, paths que se desenham, composições que se montam conforme o scroll — isso é o que separa uma LP disruptiva de um template bonito.
